# Kwalia Website & Essays

The official website for [Kwalia](https://kwalia.ai) - AI First Entertainment.

## Project Structure

```
kwalia-website/
├── index.html              # Main website
├── essays/                 # Generated essay HTML (don't edit directly)
│   └── *.html
├── content/                # Source content (edit these)
│   └── essays/
│       └── *.md           # Markdown essays with YAML frontmatter
├── templates/              # Jinja2 templates
│   ├── essay-en.html
│   └── essay-es.html
├── data/
│   ├── essays.json        # Essay metadata (auto-generated)
│   ├── books.json         # Book catalog
│   ├── fiction.json       # Fiction catalog
│   └── i18n/              # Translations
├── assets/                 # Images, PDFs
├── build_essays.py        # Build script
├── README.md              # This file
└── CLAUDE.md              # AI assistant instructions
```

## Adding a New Essay

### 1. Create Markdown file

Create a new `.md` file in `content/essays/`:

```bash
# English version
content/essays/my-new-essay.en.md

# Spanish version
content/essays/mi-nuevo-ensayo.es.md
```

### 2. Add frontmatter

Every essay needs YAML frontmatter at the top:

```yaml
---
id: my-new-essay              # Unique identifier (same for both languages)
lang: en                      # Language: en or es
slug: my-new-essay            # URL slug (filename without extension)
title: "My New Essay Title"
subtitle: "A brief description that appears below the title."
date: 2026-01-21              # Publication date (YYYY-MM-DD)
author: Javier del Puerto     # Author name
tags:                         # One or more tags
  - future
  - rights
read_time: 5                  # Optional: reading time in minutes
related:                      # Optional: IDs of related essays
  - youre-already-a-cyborg
  - the-case-for-ai-rights
book: mindkind                # Optional: mindkind or udair
translation: mi-nuevo-ensayo  # Optional: slug of translation
status: published             # published or draft
excerpt: "Short excerpt for previews and SEO."
---

Your essay content in Markdown here...
```

### 3. Write content in Markdown

The body of the file is regular Markdown:

```markdown
This is the first paragraph. It gets a drop-cap automatically.

## Section Header

Regular paragraph with **bold** and *italic* text.

External [links](https://example.com) open in new tabs automatically.
Internal [links](other-essay.html) don't.

> Blockquotes get styled with a pink left border.
```

### 4. Build

Run the build script:

```bash
python3 build_essays.py
```

This will:
- Convert your Markdown to HTML
- Apply the appropriate template (EN or ES)
- Output to `essays/`
- Update `data/essays.json`

### 5. Commit and push

```bash
git add .
git commit -m "Add new essay: My New Essay"
git push
```

## Editing Templates

Templates are in `templates/`:
- `essay-en.html` - English essay template
- `essay-es.html` - Spanish essay template

Templates use [Jinja2](https://jinja.palletsprojects.com/) syntax.

## Data Files

### essays.json

Auto-generated by `build_essays.py`. Contains metadata for all essays. Used by the essays index page.

### books.json / fiction.json

Book catalog data. Edit manually to add/update books.

### i18n/en.json / i18n/es.json

UI translations for the main website.

## Deployment

The site is hosted on GitHub Pages. Pushing to `main` automatically deploys.

## Local Development

To preview locally, use any static file server:

```bash
# Python
python3 -m http.server 8000

# Node
npx serve .
```

Then open http://localhost:8000

## Writing Guidelines

- **No AI tells**: Avoid patterns like "This is not X. This is Y.", excessive em dashes, words like "crucial", "essential", "delve", "tapestry"
- **External links**: Use full URLs starting with `https://`
- **Internal links**: Use relative paths like `other-essay.html`
- **Images**: Place in `assets/` and reference as `../assets/image.jpg`
